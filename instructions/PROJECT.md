# DTNet — Project Vision & Architecture

## What Is This Project?

DTNet is a **research prototype** for a Master's thesis at HEC Lausanne (Data Science).
It models industrial supply chains as **networks of interconnected digital twins** and uses AI to predict how disruptions cascade through the network.

**In plain terms:** instead of monitoring each factory machine individually (which is what everyone does today), we connect all the digital twins into a graph and use AI to predict the domino effect when something breaks.

## The Research Gap

Two fields exist in isolation:
1. **Digital Twins** → monitor individual machines, but in silos
2. **Supply Chain Disruption** → study how failures cascade, but with abstract models (no real machine data)

**Nobody has combined them.** That's what DTNet does.

## Three Research Questions

- **RQ1:** How to represent a supply chain as a directed graph of digital twins? (What goes on each node and edge?)
- **RQ2:** How can GNN + agent-based simulation predict cascading failures on this graph?
- **RQ3:** Does the networked approach beat isolated twins? (Spoiler: yes)

## Architecture — Three Layers

```
┌─────────────────────────────────────────────┐
│           INTELLIGENCE LAYER                │
│   GNN (PyTorch Geometric) + ABS (Mesa)      │
│   → Predicts & simulates cascading failures │
├─────────────────────────────────────────────┤
│           GRAPH LAYER                       │
│   Directed Graph G = (V, E) via NetworkX    │
│   → Nodes = digital twins, Edges = flows    │
├─────────────────────────────────────────────┤
│           NODE LAYER                        │
│   Digital Twin Agents                       │
│   → Each has sensor state + behavior rules  │
└─────────────────────────────────────────────┘
```

## Node Types (5 layers in the supply chain)

| Layer         | What it represents              | Key attributes                                    |
|---------------|----------------------------------|---------------------------------------------------|
| Supplier      | Raw material / part provider     | delivery_reliability, lead_time, capacity          |
| Logistics     | Transport / warehouse hub        | transit_time, warehouse_capacity, route_reliability|
| Plant         | Factory / production site        | production_rate, quality_rate, overall_capacity    |
| Machine       | Individual machine (digital twin)| sensors: temp, vibration, pressure, load, RPM      |
| Distribution  | Delivery to end customer         | demand_variability, fulfillment_rate, stock_level  |

## Edge Types (4 dependency types)

| Edge Type               | Meaning                                           |
|--------------------------|---------------------------------------------------|
| material_flow           | Physical material moves from A to B               |
| operational             | Plant owns/controls its machines                  |
| process_chain           | Machine A feeds Machine B on the production line  |
| shared_part_dependency  | Machines in different plants share the same part supplier |

## Data Strategy

- **Primary source:** Kaggle "Machine Demand & Failure Prediction" dataset (sensors, breakdowns, multi-entity)
- **Graph topology:** Built from dataset structure (machines × parts × plants) + synthetic supply chain nodes
- **Training data for GNN:** Generated by running thousands of disruption simulations with Mesa

## Final Deliverable

A Python prototype that:
1. Builds the DTNet graph from data
2. Runs cascading failure simulations
3. Trains a GNN to predict disruptions
4. Compares networked vs isolated twins (RQ3)
5. Visualizes everything clearly

This is a **research prototype**, not a production app. Code quality matters, but the priority is **correctness, reproducibility, and clear results** for the thesis.
